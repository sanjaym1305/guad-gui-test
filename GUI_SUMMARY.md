# GUI and Data Logging

The graphical user interface is a full-stack web application built with React 18, Material-UI, and Recharts on the frontend, backed by an Express.js REST API server connected to MongoDB Atlas for persistent data storage. The system is containerized using Docker Compose, with Nginx serving the frontend on port 80 and the Express backend running on port 5001. In direct hardware mode, the frontend communicates with the STM32 microcontroller through the Web Serial API, receiving and parsing CSV-formatted data packets at 115200 baud over USB. When no serial connection is present, the frontend automatically polls the backend API for incoming data, enabling remote monitoring through the ground station data pipeline without a direct hardware connection.

The interface displays real-time sensor data organized by packet type and hub location. It visualizes IMU gyroscope and accelerometer readings across three positions (front, center, rear) with nine values each, gap height measurements from eight inductive sensors (four lateral and four vertical), temperature readings from ten PT1000 RTD sensors distributed across the front hub (4), center hub (2), and rear hub (4), Hall effect current measurements from twelve sensors across the front hub (5), center hub (2), and rear hub (5), and battery voltages for all 144 cells arranged in an 18-by-8 grid. Semi-circle progress bars display acceleration magnitude, vertical bar indicators show gap height, and color-coded temperature grids provide rapid visual assessment of thermal conditions across all hubs.

The GUI implements the same finite state machine used by the embedded system, displaying the current pod state and providing operator controls for state transitions including initialization, health check, ready, levitation, propulsion, coasting, braking, and stopped. Safety features include an emergency stop triggered by either a dedicated GUI button or a triple-spacebar keyboard shortcut, which sends an immediate EMERGENCY_STOP command to the pod over serial. The interface monitors communication health through periodic backend health checks and displays connection status indicators for both serial and MongoDB connectivity. A timestamped console log records all incoming data, state transitions, and system events. The system also supports CSV and TXT file export through the browser File System API for offline data analysis, and all sensor data is persisted to MongoDB Atlas with timestamp and packet type indexing for historical querying by date range or sensor type.
